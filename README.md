# 1. 프로젝트 정보

## 1.1. 프로젝트 이름

- ALOT: A Life Of Travel
- 여행을 많이 가자는 의미로 제작

# 1.2. 동기

- 배낭여행을 할 때 교통 한편을 놓칠시 비행기나 고속버스를 놓치는 등의 지대한 영향이 생길 수 있음.
- 미리 계획을 작성하고 작성한 일정을 확인하면서 그런 불상사를 막을 수 있지 않을까란 생각에서 기획.

# 1.3. 팀원소개

- 권민욱(팀장): 피그마, 회원가입, 계획 및 수정 페이지, 일정(Calendar)페이지, 전체 일정페이지, 푸터 담당

- 이도현: 로그인, 메인, 체크리스트, 사용자 정보 수정 페이지, 헤더 담당

- 황운철: 상세 페이지 담당

# 4. 각 페이지 소개

## 4.1 권민욱

### 4.1.1. 회원 가입 페이지(Signup)

- 아이디, 비밀번호, 비밀번호 확인, 이메일, 이름 입력
- 유효성 검사로 한글자 입력할때마다 검사 후 조건에 맞을때까지 아래에 경고문구 표시

### 4.1.2. 계획 페이지(Plan)

- 제목, 목적지, 시작날짜, 끝날짜, 예산 입력 시 정보 DB로 전송과 동시에 tourId 부여

### 4.1.3. 일정 페이지(Calendars, EventModal)

- touId를 받아 와 계획 페이지에서 입력받은 제목과 날짜를 캘린더에 표시.
- 각 날짜 클릭시 제목, 시간, 내용, 소비금 입력 가능.

### 4.1.4 계획 수정 페이지(PlanModifiy)

- 일정 페이지에서 계획 수정하기 버튼 클릭시 이동.
- 수정 후 저장 버튼 클릭시 변경된 내용으로 DB 수정 및 일정 페이지 표시.

### 4.1.5. 전체 일정 페이지(CalendarsForAllPlan, EventModalForAll)

- 로그인한 유저가 입력한 모든 계획과 각 날짜의 일정 표시.

## 4.2 이도현

### 4.2.1. 헤더(Header)

- 홈으로 갈 수 있는 로고와 navi-bar가 있어 누르면 해당 페이지로 이동
- 로그인 되어있다면, 회원정보 또는 로그아웃을 할 수 있음
- 로그인 되어있지 않다면, navi 이동을 막고, 로그인 페이지로 강제이동 / 회원정보, 로그아웃 대신 로그인, 회원가입 표시

### 4.2.2. 로그인(login)

- localstorage에 login된 user-id를 저장하고, 이를 context에 저장하여 다른 page에서도 사용할 수 있음

### 4.2.3. 체크리스트(checklist)

- 여행 계획을 세우면, 해당 여행 고유 pk가 할당되고, 해당 pk의 여행 준비물 리스트 생성
- 사용자가 특정 물품을 입력하면, 공백/중복 여부 체크 후 리스트에 입력

### 4.2.4. 메인페이지(mainpage)

- 사용자가 등록한 여행 중 가장 빠른 일자의 계획과 d-day를 화면 상단에 표시
- 하단엔 사용자가 서비스를 이용할 마음이 들게 여행을 떠나고싶은 마음을 자극하는 이미지 첨가

### 4.2.5. 사용자 정보(userinfo)

- user의 아이디, 이름, 이메일, 가입일자를 나타내고, user가 원한다면 비밀번호 수정 기능 제공
- 비밀번호 수정 시 빈칸, 없는 비밀번호, 새 비밀번호와 현재 비밀번호 중복 검사 후, 수정이된 경우 수정일자 표기

## 4.3 황운철

### 4.3.1. 상세 페이지

#### Plan Data Get

- 계획 페이지에서 일정(여행)을 입력하고 저장했을 때, 해당 일정이 할당받은 고유 pk(tourId)를 detail 페이지의 주소에 입력하면 그 일정의 상세 정보를 로드

- 올바른 값을 입력했음에도 상세 정보를 로드하지 못하는 일부 일정이 있음

#### Plan Data PUT

- 사용자가 등록한 일정의 상세 정보를 임의로 수정

- 수정에 성공했을 경우 입력한 정보에 맞게 해당 일정의 상세 정보가 갱신됨

- 수정을 취소했을 경우 수정하기 이전의 원래 정보를 다시 로드함

- 해당 기능을 이용하기 위한 Modal 팝업창이 수정 버튼을 눌렀음에도 아무 반응이 없는 문제가 생김, 차후 개선 필요

#### Plan Data Delete

- 사용자가 등록한 일정의 상세 정보를 임의로 삭제

- 삭제에 성공하면 해당 일정의 상세 정보를 포함한 여행 계획이 바로 지워지고 메인페이지로 리다이렉트함

- 삭제를 취소했을 경우 삭제하기 이전의 원래 정보를 다시 로드함

- 해당 기능을 이용하기 위한 API 연동에서 400 에러가 발생해 실패하는 문제가 생김, 차후 개선 필요

# 5. 초기 기획과 달라진 점

## 5.1 권민욱

##### 5.1.1 일정(캘린더) 페이지, 전체 일정 페이지

- 작성한 내용을 모두 오른쪽 탭에 표시 하려 하였으나 제목과, 시간 정보 외에는 상세 페이지에만 나오는 걸로 변경

##### 5.1.2 계획 작성 페이지

- 체크리스트도 함께 추가하려 하였으나 백엔드의 테이블 문제로 체크리스트는 별도로 뺌

## 5.2 이도현

##### 5.2.1 checklist

- 기획은 checklist의 각 요소들을 수정 할 수 있도록 하려 했으나, check를 하거나 삭제하는 것이 더 효율적인데 굳이 item을 수정 할 필요는 없는 것 같아서 기능 제외

##### 5.2.2 mainpage

- 홈페이지 상단은 swiper 적용하여 전체 여행 계획들을 한 화면에서 확인할 수 있게 만드려 했으나 실패함
- main page의 하단에는 로그인이 되면 상단에 표시되는 여행의 checklist를 나타내려 했으나, 시간상의 이유로 캔슬됨

# 6. 후기

### 6.1. 권민욱

- 여러번의 변경으로 인한 손실로 기획단계가 얼마나 중요한지 깨달음.
- 백엔드와의 연동에서 필요한 부분 습득.

### 6.2. 이도현

- 일상생활 속 자주 보던 웹사이트들이 어떤식으로 기획되고 만들어졌을지 상상되는 재미있는 경험이었음

### 6.3. 황운철

- 시간 관계상 많은 기능을 추가하기보단 사용자가 보다 간편하게 일정과 계획을 관리할 수 있는 기능을 구현하는데 집중하였으나 그럼에도 만족스런 결과가 나오지 못해 상당히 아쉬움.

- 2차 프로젝트에 참여할 때는 지금보다 나은 과정과 결과물을 만들어낼 수 있도록 개인적인 노력이 많이 필요하다는 것을 체감함.
